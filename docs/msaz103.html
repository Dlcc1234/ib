Customisation spécifique du stage MSAZ103

Un dossier "Ateliers MSAZ103" a été déposé sur le bureau. Il contient surtout:
  • Les fichiers nécessaires à la réalisation des ateliers (dans des sous-dossiers nommés AZ-100.x / AZ-101.x)
  • Un lien vers le site de création des compte azure basé sur les codes fournis pour la session, nommé Azure - Validation pass
  • Un lien vers les instructions des ateliers sur Github
Le fichier readMe sur le bureau contient, pour référence éventuelle, le titre de chaque atelier, avec l'ordre conseillé de réalisation de ceux-ci, correspondant à la numérotation des modules.
Le Framework .Net 4.8 a été installé sur chaque poste,
Le module PowerShell AZ a été installé dans sa dernière version.
Le dossier Documents contient (éléments rajoutés par la commande ibSetup MSAZ103 -trainer):
  • Une présentation PowerPoint "AZ-103AIntro" pour aider à l'animation du début du stage,
  • Une présentation PowerPoint "AZ-103A-Extra-Automation" (extraite de l'ancien stage Azure M20533) pour ajouter la thématique "Azure Automation" si demande des stagiaires.

Conseils d'animation pour le stage MSAZ103

Dés le début du stage, faire créer un compte Micrsoft aux stagiaires (https://www.live.com)
Ce compte pourra ensuite être utilisé pour créer un compte Azure de test (raccourci Azure - Validation pass).
Vidéo de démonstration de la mise en place d'une connexion vNet to vNet
Prendre connaissance du fichier readme posé sur le bureau et suggérer aux stagiaires de faire de même.
En cas de soucis sur un poste, il est possible de (re)lancer la commande ibSetup MSAZ103 -force pour mettre en place ou corriger les éléments de customisation décrits précédemment.
(y ajouter l'option "-trainer" sur le poste du formateur).
Les stagiaires ont 2 supports de stage : celui du stage msaz900 et celui du stage msaz103.
Il est conseillé de mettre en avant l'intérêt du support de stage msaz900 et de prendre le temps d'en faire une sorte de synthése avant de se lancer dans l'administration Azure en tant que telle.

Atelier 7

Afin de pouvoir tester au maximum la réplication/migration de la VM, on pourra, par exemple choisir "West Europe" en région de source et "North Europe" en région de destination.
