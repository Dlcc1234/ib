remarque ateliers

Dans plusieurs ateliers, on indique de fermer un outil puis d'en agrandir la fenêtre par la suite : soit indiquer d'iconiser les outils au lieu de les fermer, soit de relancer les outils ensuite. Pour faciliter les démonstrations, le script "iscsiCluster.ps1" met en place les 3 disques partagés iSCSI nécessaires dans l'atelier/les démonstrations.

Pour l'atelier 3

Les outils DHCP et DNS peuvent également être installés dans "Roles and Features/Remote Server Administration/Tools/xxx"

Erreur goDeploy

Bien laisser le clavier en EN-US, y compris pendant les ateliers d'installation : c'est l'environnement web goDeploy qui fera la transition avec la langue du clavier du poste du stagiaire. [ATTENTION Atelier 4]

Atelier 5 Dans la partie conteneurs (exercice 2), il semblerait que le premier lancement du conteneur IIS se solde par un arrêt prématuré de celui-ci (vérifialble avec la commande docker ps), le cas échéant: La commande docker container prune permet de purger tous les conteneurs arrêtés, On pourra ensuite rappeler la commande docker run précédente pour relancer le conteneur et vérifier qu'il est stable par appel à la commande docker ps.
